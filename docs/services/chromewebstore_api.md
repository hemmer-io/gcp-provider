# Chromewebstore_api Service



**Resources**: 3

---

## Overview

The chromewebstore_api service provides access to 3 resource types:

- [Item](#item) [CRU]
- [Media](#media) [C]
- [Item](#item) [CR]

---

## Resources


### Item

Inserts a new item.

**Operations**: ✅ Create ✅ Read ✅ Update

#### Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|


#### Outputs

| Output | Type | Description |
|--------|------|-------------|
| `kind` | String | Identifies this resource as an Item. Value: the fixed string "chromewebstore#item". |
| `upload_state` | String | Status of the operation. Possible values are: - \"FAILURE\" - \"IN_PROGRESS\" - \"NOT_FOUND\" - \"SUCCESS\"  |
| `item_error` | Vec<String> | Detail human-readable status of the operation, in English only. Same error messages are displayed when you upload your app to the Chrome Web Store. |
| `public_key` | String | Public key of this item. |
| `crx_version` | String | The CRX version of the item. If the projection is draft, then it is the draft's CRX version. |
| `id` | String | Unique ID of the item. |


#### Usage Example

```kcl
# main.k
import gcp

# Initialize provider
provider = gcp.GcpProvider {
    project = "my-project-id"
}

# Create item
item = provider.chromewebstore_api.Item {
}

# Access item outputs
item_id = item.id
item_kind = item.kind
item_upload_state = item.upload_state
item_item_error = item.item_error
item_public_key = item.public_key
item_crx_version = item.crx_version
item_id = item.id
```

### Media

Upload a new package to an existing item.

**Operations**: ✅ Create

#### Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | String | ✅ | Required. Name of the item to upload the new package to in the form `publishers/{publisherId}/items/{itemId}` |



#### Usage Example

```kcl
# main.k
import gcp

# Initialize provider
provider = gcp.GcpProvider {
    project = "my-project-id"
}

# Create media
media = provider.chromewebstore_api.Media {
    name = "value"  # Required. Name of the item to upload the new package to in the form `publishers/{publisherId}/items/{itemId}`
}

```

### Item

Submit the item to be published in the store. The item will be submitted for review unless `skip_review` is set to true, or the item is staged from a previous submission with `publish_type` set to `STAGED_PUBLISH`.

**Operations**: ✅ Create ✅ Read

#### Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `publish_type` | String |  | Optional. Use this to control if the item is published immediately on approval or staged for publishing in the future. Defaults to `DEFAULT_PUBLISH` if unset. |
| `skip_review` | bool |  | Optional. Whether to attempt to skip item review. The API will validate if the item qualifies and return a validation error if the item requires review. Defaults to `false` if unset. |
| `deploy_infos` | Vec<String> |  | Optional. Additional deploy information including the desired initial percentage rollout. Defaults to the current value saved in the developer dashboard if unset. |
| `name` | String | ✅ | Required. Name of the item in the form `publishers/{publisherId}/items/{itemId}` |


#### Outputs

| Output | Type | Description |
|--------|------|-------------|
| `public_key` | String | The public key of the item, which may be generated by the store. |
| `item_id` | String | Output only. The ID of the item. |
| `name` | String | The name of the requested item. |
| `taken_down` | bool | If true, the item has been taken down for a policy violation. Check the developer dashboard for details. |
| `published_item_revision_status` | String | Output only. Status of the current published revision of the item. Will be unset if the item is not published. |
| `warned` | bool | If true, the item has been warned for a policy violation and will be taken down if not resolved. Check the developer dashboard for details. |
| `last_async_upload_state` | String | Output only. The state of the last async upload for an item. Only set when there has been an async upload for the item in the past 24 hours. |
| `submitted_item_revision_status` | String | Status of the item revision submitted to be published. Will be unset if the item has not been submitted for publishing since the last successful publish. |


#### Usage Example

```kcl
# main.k
import gcp

# Initialize provider
provider = gcp.GcpProvider {
    project = "my-project-id"
}

# Create item
item = provider.chromewebstore_api.Item {
    name = "value"  # Required. Name of the item in the form `publishers/{publisherId}/items/{itemId}`
}

# Access item outputs
item_id = item.id
item_public_key = item.public_key
item_item_id = item.item_id
item_name = item.name
item_taken_down = item.taken_down
item_published_item_revision_status = item.published_item_revision_status
item_warned = item.warned
item_last_async_upload_state = item.last_async_upload_state
item_submitted_item_revision_status = item.submitted_item_revision_status
```

## Common Operations

### Creating Multiple Resources

```kcl
import gcp

provider = gcp.GcpProvider {
    project = "my-project-id"
}

# Create multiple item resources
item_0 = provider.chromewebstore_api.Item {
}
item_1 = provider.chromewebstore_api.Item {
}
item_2 = provider.chromewebstore_api.Item {
}
```

### Conditional Creation

```kcl
# Only create in production
if environment == "production":
    item = provider.chromewebstore_api.Item {
    }
```
